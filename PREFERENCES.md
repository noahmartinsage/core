# 重要偏好设置

## GitHub 上传方式
**优先使用浏览器上传**，不使用 git push 命令。

### 原因
- 服务器到 GitHub 的网络被限制（SSL 连接中断）
- 浏览器可以通过用户的已登录会话上传
- 更稳定、更可靠

### 操作流程
1. 使用 `browser` 工具打开 GitHub 仓库页面
2. 点击 "Add file" → "Upload files"
3. 将文件复制到 `/tmp/openclaw/uploads/` 目录
4. 使用 `browser upload` 命令上传文件
5. 填写提交信息并点击 "Commit changes"

### 适用场景
- 上传新项目代码
- 更新现有文件
- 备份记忆和技能
- 任何需要推送到 GitHub 的操作
